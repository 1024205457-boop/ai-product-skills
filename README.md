# AI Product Skills

Reusable, public-safe Codex skills for AI product work: Agent design, AI search, evaluation, AIGC workflows, business automation, and data operations.

This repository is intentionally sanitized. It contains general workflows, templates, scripts, and checklists only. It does not include company source code, internal platform names, real user data, private prompts, access tokens, cookies, or production business rules.

## Skills

| Skill | Use it for |
| --- | --- |
| `agent-workflow-design` | Design controllable Agent workflows, tools, events, and fallback rules. |
| `agent-evaluation-framework` | Build evaluation dimensions, datasets, redlines, and scorecards for Agents. |
| `rag-product-design` | Design RAG, AI search, and answer-with-citation product flows. |
| `aigc-content-review-workflow` | Design generation, review, badcase, and iteration loops for AIGC products. |
| `business-process-automation-prd` | Convert manual operations into AI workflow automation PRDs. |
| `ai-product-demo-planning` | Plan portfolio-ready AI product demos for job applications. |
| `private-domain-ai-workflow` | Design private-domain AI content supply workflows. |
| `ai-image-generation-workflow` | Design batch image generation workflows with human review gates. |
| `ai-data-update-workflow` | Run safe AI-assisted data update and anomaly reporting workflows. |

## Repository Principles

- Keep each `SKILL.md` concise and executable.
- Put longer frameworks, rubrics, and templates in `references/`.
- Use `scripts/` only when deterministic automation is useful.
- Keep examples generic and fictional.
- Never publish private company details or production credentials.

## Suggested Use

Install or copy a skill folder into your Codex skills directory, then ask Codex to use the relevant skill:

```text
Use the agent-evaluation-framework skill to design an evaluation system for my shopping guide Agent.
```

For portfolio use, link this repository as evidence that your AI product work has been converted into reusable workflows rather than one-off notes.
