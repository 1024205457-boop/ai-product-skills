# 明细流水到透视表方法

读这个文件，当任务是从原始明细流水生成周报、看板、透视表或 Excel 汇总。目标不是让 AI 猜数，而是让 AI 设计并执行一条可复核的数据流水：读取原始行，清洗字段，按明确的字段映射生成 output，再回读校验。

## 1. 先写透视表契约

先把人工透视表写成契约，再写代码。契约至少包含：

```text
统计周期: 2026-07-06 ~ 2026-07-12，按本地自然日
输入表: raw_events.csv
唯一键: event_id
用户键: user_id
时间字段: event_time
行维度: event_date, source_group
列维度: event_type
指标:
- visit_users = event_type=visit 的 user_id 去重数
- click_users = event_type=click 的 user_id 去重数
- pay_users = event_type=pay 的 user_id 去重数
- click_rate = click_users / visit_users
- pay_rate = pay_users / visit_users
过滤:
- 排除 is_test=1
- 只保留 status=success
```

契约决定输入要选哪些字段。不要从“表里有哪些字段”倒推指标；要从输出需要什么指标倒推字段。

## 2. 选择输入字段的方法

字段分为 6 类。每个输出指标都必须能回到这些字段。

| 字段类型 | 作用 | 例子 | 什么时候必须选 |
| --- | --- | --- | --- |
| 唯一键 | 去重、防重复采集 | `event_id`, `order_id`, `record_id` | 需要计数记录或检查重复时 |
| 用户键 | 统计 UV、去重人数、跨表关联 | `user_id`, `account_id` | 指标是人数、UV、转化用户时 |
| 时间键 | 过滤周期、按天/周汇总 | `created_at`, `event_time`, `dt` | 任何周期报表都必须有 |
| 维度字段 | 透视表行/列分组 | `source`, `channel`, `scene`, `operator` | 输出要按来源/渠道/场景拆分时 |
| 状态字段 | 过滤和条件计数 | `event_type`, `status`, `period`, `is_test` | 指标有条件口径时 |
| 数值字段 | 求和、金额、次数 | `amount`, `duration`, `value` | 输出是 GMV、时长、次数时 |

字段选择代码模板：

```python
OUTPUT_SPEC = {
    "rows": ["event_date", "source_group"],
    "metrics": {
        "visit_users": {"filter": {"event_type": "visit"}, "field": "user_id", "agg": "nunique"},
        "click_users": {"filter": {"event_type": "click"}, "field": "user_id", "agg": "nunique"},
        "pay_users": {"filter": {"event_type": "pay"}, "field": "user_id", "agg": "nunique"},
        "revenue": {"filter": {"event_type": "pay"}, "field": "amount", "agg": "sum"},
    },
    "rates": {
        "click_rate": ("click_users", "visit_users"),
        "pay_rate": ("pay_users", "visit_users"),
    },
}

def required_fields(spec):
    fields = set(spec["rows"])
    for metric in spec["metrics"].values():
        fields.add(metric["field"])
        fields.update(metric.get("filter", {}).keys())
    return fields
```

如果要跨两张明细表生成输出，还必须声明关联键和关联粒度：

```python
JOIN_SPEC = {

    "grain": "user_id + week",
    "rule": "先在各自表内按 user_id + week 聚合，再 join，避免明细对明细造成行数膨胀",
}
```

## 3. 常见清洗动作和代码

### 3.1 清理列名、BOM 和空格

CSV 经常带 BOM、全角空格、首尾空格。第一步统一列名。

```python
import pandas as pd

def read_csv_clean(path):
    df = pd.read_csv(path, encoding="utf-8-sig")
    df.columns = (
        df.columns.astype(str)
        .str.replace("\ufeff", "", regex=False)
        .str.replace("\u3000", " ", regex=False)
        .str.strip()
    )
    return df
```

### 3.2 校验必需字段

缺字段要停止，不要让后续逻辑用空列继续跑。

```python
def require_columns(df, required, source_name):
    missing = sorted(set(required) - set(df.columns))
    if missing:
        raise SystemExit(f"{source_name} missing columns: {missing}")
```

### 3.3 标准化字符串枚举

维度字段要清理空值、空格、大小写和别名。

```python
def normalize_text(s):
    if pd.isna(s):
        return ""
    return str(s).replace("\u3000", " ").strip()

SOURCE_ALIAS = {
    "app-home": "App",
    "APP首页": "App",
    "web": "Web",
    "H5": "Web",
}

df["source"] = df["source"].map(normalize_text)
df["source_group"] = df["source"].replace(SOURCE_ALIAS)
df.loc[df["source_group"].eq(""), "source_group"] = "Unknown"
```

### 3.4 数值字段转型

不要直接 `int(row["amount"])`。原始数据可能有空字符串、逗号、百分号。

```python
def to_number(series):
    return pd.to_numeric(
        series.astype(str)
        .str.replace(",", "", regex=False)
        .str.replace("%", "", regex=False)
        .str.strip()
        .replace({"": None, "nan": None, "None": None}),
        errors="coerce",
    ).fillna(0)

df["amount"] = to_number(df["amount"])
df["duration"] = to_number(df["duration"])
```

### 3.5 布尔和状态字段转型

```python
TRUE_VALUES = {"1", "true", "True", "yes", "Y", "是"}

df["is_test"] = df["is_test"].astype(str).str.strip().isin(TRUE_VALUES)
df["status"] = df["status"].map(normalize_text)
df = df[(~df["is_test"]) & (df["status"].eq("success"))].copy()
```

### 3.6 去重

先明确去重规则。常见规则：

- 明细事件：按 `event_id` 去重。
- 用户状态快照：按 `user_id + date` 保留最后更新时间。
- 订单：按 `order_id` 去重，不要按用户去重。

```python
df = df.drop_duplicates(subset=["event_id"], keep="last")

df = (
    df.sort_values("updated_at")
    .drop_duplicates(subset=["user_id", "event_date"], keep="last")
)
```

### 3.7 标签拆分和展开

标签列可能是逗号、顿号、分号混用。先统一分隔符，再展开。

```python
tag_rows = df[["event_date", "source_group", "user_id", "tags"]].copy()
tag_rows["tag"] = (
    tag_rows["tags"].fillna("").astype(str)
    .str.replace("、", ",", regex=False)
    .str.replace(";", ",", regex=False)
    .str.split(",")
)
tag_rows = tag_rows.explode("tag")
tag_rows["tag"] = tag_rows["tag"].map(normalize_text)
tag_rows = tag_rows[tag_rows["tag"].ne("")]

tag_pivot = (
    tag_rows.groupby(["event_date", "source_group", "tag"], dropna=False)
    .agg(tag_count=("tag", "size"), tag_users=("user_id", "nunique"))
    .reset_index()
)
```

### 3.8 空值处理

空值不能一律填 0。按字段类型处理：

| 字段 | 处理方式 |
| --- | --- |
| 唯一键、用户键、时间键 | 空值通常应报错或进入异常表 |
| 维度字段 | 可填 `Unknown`，但要在质量检查里记录数量 |
| 数值字段 | 可填 0，但要确认空值是否代表 0 |
| 分母字段 | 分母为 0 时比率写 0 或空值，必须统一规则 |

```python
quality = []
for col in ["event_id", "user_id", "event_time"]:
    n = df[col].isna().sum() + df[col].astype(str).str.strip().eq("").sum()
    if n:
        quality.append({"check": f"empty_{col}", "value": int(n), "level": "FAIL"})

df["source_group"] = df["source_group"].replace("", "Unknown")
df["amount"] = to_number(df["amount"])
```

## 4. 时间和时区处理

时间问题是周报最常见的错源。原则：

1. 如果业务按本地自然日统计，就用本地时区生成日期，不要用 UTC 日期。
2. JavaScript 不要用 `toISOString().slice(0, 10)` 当本地日期，因为它会转 UTC。
3. 过滤周期建议用半开区间：`start <= time < end + 1 day`。
4. 如果 API 参数是日期字符串，按接口口径传 `YYYY-MM-DD`；如果是毫秒时间戳，明确是本地零点还是 UTC 零点。

### 4.1 JavaScript 周范围，不用 toISOString

```javascript
function getWeekRange(dateStr) {
  const d = new Date(dateStr + 'T00:00:00');
  const day = d.getDay();
  const diffToMonday = day === 0 ? -6 : 1 - day;
  const monday = new Date(d);
  monday.setDate(d.getDate() + diffToMonday);
  const sunday = new Date(monday);
  sunday.setDate(monday.getDate() + 6);
  const fmt = d => `${d.getFullYear()}-${String(d.getMonth()+1).padStart(2,'0')}-${String(d.getDate()).padStart(2,'0')}`;
  return { start: fmt(monday), end: fmt(sunday) };
}
```

### 4.2 Python 本地日期解析

```python
def parse_local_time(series, source_tz="Asia/Shanghai"):
    dt = pd.to_datetime(series, errors="coerce")
    if getattr(dt.dt, "tz", None) is None:
        dt = dt.dt.tz_localize(source_tz)
    else:
        dt = dt.dt.tz_convert(source_tz)
    return dt

df["event_time"] = parse_local_time(df["event_time"])
df["event_date"] = df["event_time"].dt.strftime("%Y-%m-%d")
```

### 4.3 半开区间过滤

```python
def filter_period(df, time_col, start, end, tz="Asia/Shanghai"):
    start_ts = pd.Timestamp(start, tz=tz)
    end_exclusive = pd.Timestamp(end, tz=tz) + pd.Timedelta(days=1)
    return df[(df[time_col] >= start_ts) & (df[time_col] < end_exclusive)].copy()
```

### 4.4 Excel 日期序列兼容

Excel 里日期可能是 `datetime`、`date`、数字序列或字符串。写入/查找要兼容。

```python
from datetime import datetime, date, timedelta

def date_to_serial(d):
    return (d - date(1899, 12, 30)).days

def match_date(val, target):
    if val is None:
        return False
    if isinstance(val, datetime):
        return val.date() == target
    if isinstance(val, date):
        return val == target
    if isinstance(val, (int, float)):
        return (datetime(1899, 12, 30) + timedelta(days=int(val))).date() == target
    s = str(val).strip()
    for fmt in ["%Y-%m-%d", "%m-%d"]:
        try:
            d = datetime.strptime(s, fmt).date()
            return d.month == target.month and d.day == target.day
        except ValueError:
            pass
    return False
```

## 5. 从输入到输出的字段映射

用配置表写清楚映射，而不是把列号散落在代码里。

```python
METRIC_SPEC = {
    "visit_users": {
        "filter": lambda d: d["event_type"].eq("visit"),
        "field": "user_id",
        "agg": "nunique",
        "output_col": "visit_users",
    },
    "click_users": {
        "filter": lambda d: d["event_type"].eq("click"),
        "field": "user_id",
        "agg": "nunique",
        "output_col": "click_users",
    },
    "pay_users": {
        "filter": lambda d: d["event_type"].eq("pay"),
        "field": "user_id",
        "agg": "nunique",
        "output_col": "pay_users",
    },
    "revenue": {
        "filter": lambda d: d["event_type"].eq("pay"),
        "field": "amount",
        "agg": "sum",
        "output_col": "revenue",
    },
}

RATE_SPEC = {
    "click_rate": ("click_users", "visit_users"),
    "pay_rate": ("pay_users", "visit_users"),
}
```

聚合函数：

```python
def aggregate_metric(group, full_df, spec):
    rows = full_df.loc[group.index]
    mask = spec["filter"](rows)
    field = spec["field"]
    if spec["agg"] == "nunique":
        return rows.loc[mask, field].nunique()
    if spec["agg"] == "sum":
        return rows.loc[mask, field].sum()
    if spec["agg"] == "count":
        return rows.loc[mask, field].count()
    raise ValueError(f"unsupported agg: {spec['agg']}")

def safe_rate(numerator, denominator):
    if denominator is None or denominator == 0:
        return 0
    return round(numerator / denominator, 4)
```

生成输出：

```python
def build_summary(df):
    grouped = df.groupby(["event_date", "source_group"], dropna=False)
    rows = []
    for keys, group in grouped:
        out = {"event_date": keys[0], "source_group": keys[1]}
        for name, spec in METRIC_SPEC.items():
            out[spec["output_col"]] = aggregate_metric(group, df, spec)
        for rate_name, (num, den) in RATE_SPEC.items():
            out[rate_name] = safe_rate(out[num], out[den])
        rows.append(out)
    return pd.DataFrame(rows)
```

## 6. 透视表生成代码模板

适合明细表到多个工作表输出。

```python
import pandas as pd

REQUIRED = {
    "event_id", "user_id", "event_time", "source", "event_type",
    "status", "is_test", "amount",
}

def clean_events(path, start, end):
    df = read_csv_clean(path)
    require_columns(df, REQUIRED, "events")

    df["event_time"] = parse_local_time(df["event_time"])
    df = filter_period(df, "event_time", start, end)
    df["event_date"] = df["event_time"].dt.strftime("%Y-%m-%d")

    df["source"] = df["source"].map(normalize_text)
    df["source_group"] = df["source"].replace(SOURCE_ALIAS).replace("", "Unknown")
    df["event_type"] = df["event_type"].map(normalize_text)
    df["status"] = df["status"].map(normalize_text)
    df["is_test"] = df["is_test"].astype(str).str.strip().isin(TRUE_VALUES)
    df["amount"] = to_number(df["amount"])

    df = df[(~df["is_test"]) & df["status"].eq("success")].copy()
    df = df.drop_duplicates(subset=["event_id"], keep="last")
    return df

def build_event_type_pivot(df):
    pivot = pd.pivot_table(
        df,
        index=["event_date", "source_group"],
        columns="event_type",
        values="user_id",
        aggfunc=pd.Series.nunique,
        fill_value=0,
    )
    pivot.columns = [str(c) for c in pivot.columns]
    return pivot.reset_index()

def build_quality_checks(raw, cleaned):
    return pd.DataFrame([
        {"check": "raw_rows", "value": len(raw), "level": "INFO"},
        {"check": "cleaned_rows", "value": len(cleaned), "level": "INFO"},
        {"check": "duplicate_event_id", "value": int(cleaned["event_id"].duplicated().sum()), "level": "FAIL"},
        {"check": "date_min", "value": cleaned["event_date"].min() if len(cleaned) else "", "level": "INFO"},
        {"check": "date_max", "value": cleaned["event_date"].max() if len(cleaned) else "", "level": "INFO"},
        {"check": "unknown_source_rows", "value": int(cleaned["source_group"].eq("Unknown").sum()), "level": "WARN"},
    ])

def write_output(input_path, output_path, start, end):
    raw = read_csv_clean(input_path)
    cleaned = clean_events(input_path, start, end)
    summary = build_summary(cleaned)
    event_pivot = build_event_type_pivot(cleaned)
    checks = build_quality_checks(raw, cleaned)

    with pd.ExcelWriter(output_path) as writer:
        summary.to_excel(writer, sheet_name="summary", index=False)
        event_pivot.to_excel(writer, sheet_name="event_type_pivot", index=False)
        checks.to_excel(writer, sheet_name="quality_checks", index=False)
```

## 7. 写入 Excel 和回读校验

如果 output 要写进已有 Excel，不要依赖固定行号。用日期和表头定位。

```python
def find_row_by_date(ws, date_col, target, start_row):
    for r in range(start_row, ws.max_row + 1):
        if match_date(ws.cell(row=r, column=date_col).value, target):
            return r
    return None

def find_col_by_header(ws, header_row, header_name):
    for c in range(1, ws.max_column + 1):
        if str(ws.cell(row=header_row, column=c).value).strip() == header_name:
            return c
    raise KeyError(f"header not found: {header_name}")
```

写入后要回读：

```python
def write_and_readback(ws, row, metric_to_value, header_row=1):
    for metric, value in metric_to_value.items():
        col = find_col_by_header(ws, header_row, metric)
        ws.cell(row=row, column=col, value=value)

    mismatches = []
    for metric, expected in metric_to_value.items():
        col = find_col_by_header(ws, header_row, metric)
        actual = ws.cell(row=row, column=col).value
        if actual != expected:
            mismatches.append((metric, expected, actual))
    if mismatches:
        raise SystemExit(f"read-back mismatch: {mismatches}")
```

插入列后，如果旧公式引用被 Excel 库移动错，要修复公式引用：

```python
import re
from openpyxl.utils import get_column_letter

def fix_formulas_after_insert(ws, insert_col, max_row, max_col):
    for c in range(insert_col + 1, max_col + 2):
        for r in range(1, max_row + 1):
            v = ws.cell(row=r, column=c).value
            if isinstance(v, str) and v.startswith("="):
                new = v
                for old_c in range(max_col, insert_col - 1, -1):
                    old_letter = get_column_letter(old_c)
                    new_letter = get_column_letter(old_c + 1)
                    new = re.sub(f"(?<![A-Z]){old_letter}(?=\\d)", new_letter, new)
                if new != v:
                    ws.cell(row=r, column=c).value = new
```

## 8. AI 边界

AI 应该做：

- 根据输出反推输入字段。
- 把字段映射、过滤条件、分母口径写成可读配置。
- 生成清洗脚本、透视脚本、质量检查和回读校验。
- 对异常下降、空值、重复、分母变小做机械检查。
- 解释“从现有数据能看出的原因”和“需要业务确认的原因”的边界。

AI 不应该做：

- 猜缺失字段或猜枚举值。
- 把空值直接当成 0，除非字段规则明确。
- 用旧 JSON、旧导出或旧任务结果冒充本次采集。
- 在没有确认分母口径时计算转化率、完成率、占比。
- 把业务结论写成事实，除非原始数据能直接支持。
- 跳过回读校验或异常检查。
